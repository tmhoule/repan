Repan Documentation

A team task management system built for managers who need visibility into workload, capacity, and delivery health.

Key Concepts

Tasks

Trackable work items with priority, effort, due date, and progress

Boulders

Ongoing commitments tracked by time allocation (e.g., on-call at 20%)

Backlog

Queue of unassigned tasks for team members to claim

Buckets

Manager-defined categories for organizing work

Todos

Personal reminders with no tracking or deadlines

Architecture

Repan is a Next.js application backed by PostgreSQL via Prisma ORM. It runs in Docker with two containers: the app server and the database. A daily cron job handles notifications, archiving, and workload snapshots.

Getting Started

Install, configure, and start using Repan in minutes.

Running with Docker

docker compose up -d --build

This starts two containers:

  • db — PostgreSQL 16 on port 5432
  • app — Repan on port 3000

The app automatically waits for the database, runs migrations, and starts the server. Access it at http://localhost:3000.

First-Time Setup

On first launch, Repan shows a setup screen:

  1. Enter a display name and password (min 6 characters)
  2. Click Create Account
What gets created: Your super admin user, a "Default Team" with you as the manager, and the default achievement badges.

Production Deployment

docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d

Set these environment variables:

VariablePurpose
SESSION_SECRETStrong random string for signing session cookies
DISABLE_SECURE_COOKIESSet to "true" only if not behind HTTPS
CRON_SECRETBearer token for the daily cron job

The app listens on 127.0.0.1:3000. Use a reverse proxy (nginx, Caddy) for HTTPS.

Adding Team Members

  1. Log in as a manager or super admin
  2. Open the Admin panel (user menu, top right)
  3. Click Create User
  4. Enter their name, set a password, assign to a team
  5. They can now log in and start working

Daily Cron Job

Set up an external scheduler to call this once daily:

POST /api/cron
Authorization: Bearer YOUR_CRON_SECRET

The job archives old tasks, purges notifications, sends due-date reminders, weekly risk digests, and records workload snapshots.

Tasks & Workflows

Everything about creating, managing, and completing work in Repan.

Task Types

Standard Tasks

The core work item. Each task has:

  • Title (required) and optional Description
  • Priority — High, Medium, or Low
  • Effort Estimate — Small, Medium, or Large
  • Due Date — Optional deadline
  • Progress — 0–100% slider
  • Assignment — Who’s responsible
  • Bucket — Optional category

Boulders

Ongoing operational commitments with no end date (e.g., on-call rotation, code review duty). Tracked by time allocation (0–100% of a person’s time) instead of progress. They appear in a dedicated “Boulders” section and factor into workload calculations.

Todos

Personal quick reminders with no tracking, priority, or due date. Completing a todo awards 1 point.

Task Statuses

Not Started In Progress Blocked Stalled Paused Done Boulder

Task Lifecycle

Not Started → In Progress ↔ Blocked Stalled Paused → Done
  • Moving to In Progress records when work started (used for cycle time)
  • Moving to Blocked requires a reason and notifies the assignee
  • Resolving a block notifies the assignee and awards 5 points
  • Moving to Done awards points based on effort and checks on-time streaks
  • Completed tasks can be reopened from the History page

Tasks Page Layout

  1. Active Tasks — Your in-progress and not-started tasks
  2. To Dos — Personal reminders (collapsible)
  3. Stalled / Blocked / Paused — Tasks needing attention
  4. Boulders — Ongoing commitments
  5. Completed — Done tasks (collapsed by default)
  6. Backlog — Unassigned tasks available to claim

Task Cards

Each card shows title, status and priority badges, staleness indicator, progress slider, due date, and quick actions: Flag (change status), Comment (inline), and Done.

Flag Menu

Quickly change a task’s status:

  • Mark Blocked / Stalled / Paused / In Progress
  • Reset to Not Started (unclaimed tasks only)
  • Move to Backlog
  • Delete Task (creator, assignee, or manager)

Creating Tasks

Click Create Task and fill in title, description, bucket, priority, effort, and due date. Tasks are assigned to you by default. Managers can reassign to other team members.

Editing & Comments

Click any task title to open the detail page. Changes auto-save with an 800ms debounce. The detail page shows the full activity log and comment thread. Comments award 1 point (max 5/day).

Backlog & Planning

Managing the queue of unassigned work and forecasting delivery.

The Backlog

The backlog is the queue of unassigned tasks at the bottom of the tasks page. Tasks enter the backlog when created without an assignee, moved there via the Flag menu, or when a deactivated user’s tasks are returned.

Backlog Health

MetricWhat it shows
Total ItemsNumber of unassigned tasks
Total EffortSum of effort points (S=1, M=3, L=5)
Estimated WeeksTime to clear at current throughput
TrendGrowing, shrinking, or stable vs last week

Claiming Tasks

Click Claim on any backlog task to assign it to yourself. This awards 3 points and is atomic — if two people click simultaneously, only one succeeds.

Backlog Ordering

Tasks are sorted by urgency combining:

  • Priority weight — High-priority tasks float to the top
  • Due date proximity — Approaching deadlines rank higher
  • Effort modifier — Larger tasks with deadlines get a bump
  • Age — Gradual urgency boost over time (+1/week, up to +10)

Buckets

Manager-defined categories for organizing work. Each bucket has a name and color. Use the filter bar to view tasks by bucket or “Uncategorized”.

Forecasting

Each backlog item shows estimated weeks until someone starts working on it, calculated from weekly throughput and backlog position. Updates as throughput changes.

Teams & Roles

How access control, team structure, and multi-team support work.

Roles

RoleAccess
Super AdminAdmin panel, SSO config, session settings, all teams, can grant super admin
ManagerDashboard, Reports, Capacity, Teams Overview, bucket management, task assignment, contribution table
StaffOwn tasks, backlog claiming, team profiles (read-only), History, Standup
Staff assignment: Staff cannot change task assignment by default. Managers can enable this per team via “Users can assign tasks to others” in Admin settings.

Team Structure

Users belong to teams via memberships. Each membership has a team-level role (Manager or Member). A user can be a manager on one team and a member on another.

Multi-Team Support

Users with multiple teams see a team switcher in the user menu. The active team scopes all data. After login, single-team users go directly to tasks; multi-team users see the team selection page.

Streaks

  • Daily Check-in — Any action increments it; resets after a full day of inactivity
  • Weekly Momentum — Complete a task each week; resets after a week with no completions
  • On-Time Completion — Complete before due date; resets on a miss

Average days from started to completed by effort size. Expected ratios: medium ≈ 3× small, large ≈ 5× small.

Capacity Planning

Forward-looking view of who has room for more work.

How Load is Calculated

Each person’s load is a weighted percentage where 100% = fully utilized:

1. Priority Weights

PriorityDefault Weight
High60%
Medium35%
Low10%

2. Status Multipliers

StatusMultiplierExample
Active100%High-priority = 60%
Blocked5%High-priority blocked = 3%
Stalled / Paused25%High-priority stalled = 15%

3. Remaining Work

A task at 80% complete contributes only 20% of its weighted value.

4. Boulder Allocation

Added directly — a 20% on-call boulder eats 20% capacity.

5. Week Scoping

Tasks split by due date: This Week (next 7 days) and Next Week (days 8–14). Undated tasks shown separately.

Color Coding

Green: < 80% — has capacity Amber: 80–100% — near capacity Red: > 100% — overcommitted

Tips

Before assigning: Check who has green capacity. Boulders eat capacity — a 40% boulder leaves only 60% for task work. Undated tasks are hidden load not reflected in the percentages.

Admin Guide

User management, SSO, session settings, and team configuration.

User Management

Creating Users

  1. Click Create User
  2. Enter display name and password (or leave blank for SSO)
  3. Choose team(s)
  4. Optionally mark as SSO user

Deactivating Users

When deactivated: login blocked, active tasks return to backlog, boulders archived, history preserved.

Team Settings

Priority Weights

Configure how priorities contribute to workload. Higher values = more load per task of that priority. Used in Dashboard, Capacity, and Reports.

Status Multipliers

Blocked (default 5%) and Stalled (default 25%) reduce a task’s contribution to load since it can’t be actively worked.

Permissions

Users can assign tasks to others — Off by default. When enabled, any member can reassign tasks.

Badge Management

Create custom badges with criteria types: count_action, streak_milestone, consecutive_action, single_day_count, total_points, or compound (AND/OR).

SSO / SAML

  1. Go to SSO tab in Admin
  2. Enter your App URL
  3. Paste IdP metadata URL or enter details manually
  4. Click Save & Enable SSO

Configure your IdP with:

SettingValue
ACS URLhttps://your-url/api/auth/saml/callback
SP Entity IDShown in SSO settings
NameID formatPersistent or email
JIT Provisioning: First-time SSO users are auto-created as staff with no team. A manager must add them to a team.

Session Configuration

Set idle timeout (15 min – 30 days, default 6 hours). Sessions refresh on activity. Client polls every 5 minutes to detect expiry.

Daily Cron Job

POST /api/cron
Authorization: Bearer YOUR_CRON_SECRET

Performs: archive old tasks, purge notifications, due-date reminders, backlog alerts, weekly risk digests, workload snapshots.
